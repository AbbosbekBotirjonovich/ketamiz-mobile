# Handing the project over to a new owner (transfer guide)

This document describes how to fully transfer the `ketamiz-mobile` project to
another GitHub account and what needs to be reconfigured afterwards.

## C. Reconfigure AFTER the transfer (new owner)

### 1. Re-enable Actions
`new-owner/ketamiz-mobile` â†’ Actions tab â†’ enable workflows if prompted
("I understand my workflows...").

### 2. Secrets â€” VERIFY (kept on transfer, but confirm)
Settings â†’ Secrets and variables â†’ Actions. There should be 11:

**Android (5):**
- `KEYSTORE_BASE64`, `STORE_PASSWORD`, `KEY_PASSWORD`, `KEY_ALIAS`,
  `SERVICE_ACCOUNT_JSON`

**iOS (6):**
- `MATCH_PASSWORD`, `MATCH_GIT_URL`, `MATCH_SSH_PRIVATE_KEY`,
  `APP_STORE_CONNECT_KEY_ID`, `APP_STORE_CONNECT_ISSUER_ID`,
  `APP_STORE_CONNECT_KEY_P8`

If any were lost on transfer (rare), re-add them.

### 3. Update `MATCH_GIT_URL` âš ď¸Ź
The certs repo is transferred too, so its URL changes:
- Old: `git@github.com:AbbosbekBotirjonovich/ketamiz-ios-certs.git`
- New: `git@github.com:NEW-OWNER/ketamiz-ios-certs.git`

Update the `MATCH_GIT_URL` secret to the new URL.

### 4. iOS deploy key â€” reconfigure âš ď¸Ź
The current deploy key is bound to the old certs repo. After transfer:
- New `ketamiz-ios-certs` (under the new owner) â†’ Settings â†’ Deploy keys
- Generate a new read-only deploy key pair:
  `ssh-keygen -t ed25519 -f ci_deploy -N ""`
- Add the PUBLIC part to the certs repo Deploy keys (do NOT allow write access)
- Set the PRIVATE part as the `MATCH_SSH_PRIVATE_KEY` secret

### 5. Signing accounts
- **iOS**: the Apple Developer account (`rustamovramziddin7`, Team `DJ6Z29BM4D`)
  must belong to the new owner, or the new owner must be invited to that team.
- **Android**: Play Console + keystore. The new owner needs access to the Play
  Console; `SERVICE_ACCOUNT_JSON` is tied to that Play account.

## D. Post-transfer smoke test

1. **CI**: push a small commit and confirm CI is green.
2. **Android**: push a new `vX.Y.Z` tag â†’ confirm it uploads to Play.
3. **iOS**: push a new `ios-vX.Y.Z` tag â†’ confirm it uploads to TestFlight.

## E. Securely hand over the secrets

Give the new owner the following over a secure channel (password manager,
encrypted message) â€” CI/CD won't work without them:
- Android keystore file (`upload-keystore.jks`) + passwords
- iOS Match passphrase (`MATCH_PASSWORD`)
- App Store Connect API `.p8` file + Key ID + Issuer ID
- (The deploy key is generated by the new owner; no need to hand it over.)
